    UseSafeSchemaLoad

    Specify stored procedure or table function schema load mode. When false, procedure or function will be executed with option. Otherwise more safe approach will be used (currently supported only by SQL Server and uses sp_describe_first_result_set stored procedure). If safe-load mode not supported by database, schema load will be disabled. While is safe in most of cases, it could create issues, when executed procedure contains non-transactional logic.
